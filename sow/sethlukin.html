Web destacada

Seth Lukin

Selección



· He seleccionado este sitio porque es un portafolio en el cual destaca el diseño a parte de sus trabajos, que combina trabajo de UX/UI, motion, web y visual.

· Me resulta un excelente ejemplo para estudiar desde la perspectiva de diseño web — no solo contenido estático, sino también interacción, animaciones, estructura narrativa del portafolio — lo que lo hace muy relevante para analizar y tomar como ejemplo para mi propio portafolio.

Impresión inicial



· Al llegar al sitio se siente modernidad y experimentación. Se percibe que no es un portafolio típico estático, sino uno que quiere impresionar, generar impacto, mostrar habilidades de animación e interacción. Demuestran profesionalismo y originalidad.

Estructura



La estructura del sitio sigue una organización típica de portafolio:

· Header / Intro: Una presentación central que indica nombre, rol, localización, contacto… y prosigue una breve introducción con elementos decorativos que los usa para dividir las páginas a medida que bajas.

· Sección de trabajos: Se muestran los proyectos destacados divididos por secciones temáticas: “UX / UI”, “Web”, “Motion”, “Visual”.

· Casos de estudio: Cada proyecto lleva a una página donde se expone el problema, proceso, imágenes, detalles de investigación, resultados.

· Contacto: En varias partes se invita a “Say hi”, “Start a project”.

· Footer: Calls to action, su ubicación y hora local, fecha de última actualización.

· No hay un menú complejo con muchas subsecciones: la navegación es relativamente sencilla, lo que favorece la exploración del contenido visual.

Visual



· Paleta de colores: Predominan blancos (#EAE9E9) de fondo junto con negro (#040404). Además hay acentos de color en algunos elementos decorativos. La limitada paleta de colores permite que las piezas de trabajo destaquen.

· Tipografía: Uso de tipografías sans serif modernas y bold, con buena jerarquía: títulos grandes para impacto visual, cuerpo de texto legible.

· Branding: El diseñador se posiciona como “especialista creativo / interactivo” más que como “agencia genérica”. El estilo visual lo comunica: es minimalista en pero rico en animación e interacción. Hay coherencia en la voz (“I design interactive experiences…”).

· Imágenes: Sigue un estilo visual de pixel art, personajes hechos a partir de píxeles, transiciones en las que las imagenes aparecen con degradados de pixeles, incluso el cursor. Está todo muy unificado y sigue un estilo gráfico coherente con sus trabajos presentados.
En conjunto, el aspecto visual es muy fuerte, pensado para destacar no solo el trabajo sino también la habilidad técnica en interacción.

Interacción



· Al hacer scroll, se perciben animaciones de entrada de elementos con efecto reveal, páginas que hacen slide in a medida que haces scroll.

· Microinteracciones: por ejemplo efectos al pasar el ratón sobre elementos como imágenes, textos, botones y estos se amplían sutilmente.

· Elementos decorativos del fondo, animados en bucle, no muy estridentes para no captar demasiada atención, pero que añaden valor visual.

· El flujo de navegación no es estático, se siente dinámico y con “movimiento” que añade valor visual.

Usabilidad



· Claridad: Los objetivos principales son claros: quién es el diseñador, qué hace, mostrar su trabajo, invitar a contacto. Navegar hacia los proyectos es evidente.

· Accesibilidad: Al ser un sitio visual y animado, los efectos pueden afectar a usuarios sensibles (movimiento, desplazamiento rápido), se debe asegurar opción de reducir movimiento.

· Carga: Dada la complejidad visual y animación, el sitio puede tener carga pesada, se ralentiza a medida que pasas tiempo en la web desde el ordenador. En el móvil parece ir mejor.

· Responsive / móvil: El diseño se adapta, aunque los textos y botones quedan en un formado demasiado grande.

Técnico


Velocidad:
El sitio presenta una buena velocidad.

Semántica:
La estructura del código HTML está bien organizada mediante el uso de etiquetas semánticas, como encabezados (h1, h2) y secciones (section, header, footer).

Core Web Vitals:
Las métricas de Core Web Vitals están bien y ofrecen una experiencia de usuario fluida.

Criterios de excelencia




· Creatividad: Muy elevado. No es un portafolio estándar sino uno que aprovecha la web como medio creativo: interacción, animación, scroll, micro-movimientos. Empieza demostrando sus habilidades desde el segundo 1 al que accedes a su web.

· Contenido: Fuerte. Los casos de estudios muestran proceso, reflexión, análisis… Bien desarrollado y explicado, coherente entre todos los contenidos.

· UX: Buena. Aunque estilizado, la navegación es coherente, el público puede entender rápidamente quién es, qué hace y cómo contactar. Los elementos colocados y ordenados en sus respectivos sitios. La experiencia de usuario es memorable, lo que es adecuado para un portafolio de diseño.

Conclusión

· Puntos fuertes:
Estilo visual muy distintivo, que comunica la identidad del diseñador. Interacciones y animaciones bien implementadas, que refuerzan el mensaje. Buena estructura narrativa en los trabajos: no solo “ver”, sino “entender”. Branding personal coherente: “Diseñador de experiencias interactivas que rompen patrones”.

· Debilidades:
Bajo rendimiento en dispositivos menos potentes. Es un sitio muy “premium” — quizá menos ideal si el usuario busca información rápida sin animaciones; para algunos públicos puede sentirse más “show”. Si el contenido crece mucho, la gestión de carga de nuevos trabajos/animaciones puede hacerse intensiva.

· Aprendizajes:
Un portafolio de diseño se beneficia enormemente de usar la propia web como medio de demostración, lo hace ser menos plano, más interactivo y funcional. El equilibrio entre “experiencia memorable” y “usabilidad clara” es clave: este sitio lo logra bastante bien.
